# Pre-Launch Follow-Ups

The original pre-launch follow-up tracker mixed public engineering notes with
private event, store and operations planning. It has been replaced with this
public-safe note.

Current public limitations are summarized in [`README.md`](../README.md). Keep
private launch blockers, account checks, reviewer access and event-specific
follow-ups outside this repository.

One product follow-up remains intentionally coupled: if direct native organizer
account sign-in is added, password and the web app's OAuth providers must ship
together. Until that parity exists, the app exposes only one-time QR device
pairing and does not present a password-only login screen.
