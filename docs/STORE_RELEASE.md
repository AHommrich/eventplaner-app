# Store Release Notes

Store metadata, reviewer credentials, console account state and submission
history are intentionally not tracked in the public repository.

For a public-safe build checklist, see [`docs/RELEASE.md`](RELEASE.md).

Before a real store submission, keep the private checklist up to date with:

- Apple Developer and Play Console account access.
- Bundle identifier and package name ownership.
- Support, privacy and imprint URLs.
- Reviewer access instructions with demo credentials.
- Age rating and data-safety questionnaire answers.
- Content moderation and support-response process.

Never commit real reviewer tokens, store account identifiers or private console
screenshots.
